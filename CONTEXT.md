# 쿠마에이전시 — 프로젝트 컨텍스트

> 이전 대화(journal 폴더 세션)에서 이어옴. 2026-06-23 기준.

## 사업 개요

- **회사명**: 쿠마에이전시 (가칭 검토했던 후보: AEO Lab/스튜디오 — 폐기, 비전문가 고객에게 의미 전달 안 됨)
- **기존 사업자**: "꿈을담은에이전시" (유령회사, 사업자만 살아있음, 업종에 홈페이지 제작업 등록되어 있음). 이름 변경 검토 중 — 세무서 정정 절차 필요, 시점 미정.
- **핵심 사업**: 성형외과를 시작으로 메디컬(피부과/치과/한의원) + 중소기업/전문직(변호사/세무사/노무사) 대상 **SEO + AI 검색 최적화(GEO/AEO) 기반 홈페이지 제작 대행**

## 배경 / 시작 계기

- 기존에 운영하던 "우쭈쭈"(펫리뷰의 피벗 버전) 프로젝트가 컨셉이 복잡해지고 맘에 안 들어서 중단 → 브레인스토밍으로 새 아이템 탐색
- 본인 직업: 로코코성형외과 마케팅 담당, 로코코 저널(journal 폴더, rococo-journal repo)을 SEO/AI검색 최적화 구조로 직접 구축한 경험 보유
- 펫리뷰에서 배운 교훈: "콘텐츠 없이 플랫폼 기능만 쌓으면 망한다" — 이번 사업은 처음부터 포트폴리오(콘텐츠)부터 만들고 시작

## 시장 분석 (Gemini 분석 + 자체 논의 종합)

- 국내 의원급 홈페이지 대부분 "이미지 통짜" 제작 → 구글/AI 봇이 텍스트를 못 읽어 검색 누락
- 노후화된 웹표준, 모바일 미대응, 구조화 데이터(Schema.org) 부재 → AI 검색(ChatGPT/Perplexity)에 인용 안 됨
- 환자 질문 중심(Q&A/FAQ) 구조 부재 → AEO/GEO 최적화 안 되어 있음
- 확장 가능 시장: 치과/한의원(로컬SEO), 변호사/세무사(신뢰형 Q&A), 소상공인(가성비 원페이지)

## 의사결정 사항

### 타겟 & 포트폴리오
- 가짜 클라이언트명 대신 **쿠마에이전시 자체 브랜드로 포트폴리오 샘플 제작** 결정
- 디자인 컨셉 4가지 방향 (순서대로 1개씩 제작 예정):
  1. **미니멀 메디컬형** (성형외과/피부과) — 화이트+톤다운 포인트컬러, 텍스트/신뢰신호 우선
  2. **프리미엄 비주얼형** (피부과/한방병원) — 사진 풍부하되 alt텍스트/구조화데이터로 SEO 유지
  3. **신뢰/권위형** (변호사/세무사) — 다크네이비+골드, Q&A 스키마로 승소사례 구조화
  4. **로컬 가성비형** (소상공인/학원) — 원페이지, 모바일퍼스트, 로컬SEO

### 운영 모델
- 콘텐츠: 병원 기존 자료 전달받거나 업로드 포털 구축 → 이미지 속 텍스트(OCR 어려운 경우)는 본인이 수동 처리
- **과금 구조 (확정)**: 구축비(1회, 초기 수정 N회 포함) + 월 유지비(선택, 호스팅/도메인/SEO 모니터링만, 콘텐츠 수정은 별도 과금)
- **절대 하지 말 것**: "검색노출 보장" 약속. 등록(Search Console 등)은 1회성 작업이지만 실제 순위 상승은 보장 불가 — 계약서에 명시 필요
- 유지보수 고객에게 분기별 Search Console 데이터(노출수/클릭수/게재순위) 리포트 제공 (무료 데이터, 신뢰 유지용)
- 해외환자 대상 다국어 페이지도 옵션으로 제공 가능 (journal 폴더에 이미 다국어 구현 경험 있음)

## 핵심 영업/가격 전략 (중요)

### "세컨 홈페이지" 포지셔닝
- 기존 홈페이지(이미지/브랜딩 위주) vs 새로 만드는 사이트(검색엔진·AI검색 노출 전용)를 **명확히 구분해서 설명**해야 함. 안 그러면 고객이 "왜 두 개 필요하냐" 혼란스러워함.
- 멘트 기준: 기존 사이트 = 브랜딩/예약전환용, 새 사이트 = 검색엔진+AI검색(ChatGPT/Perplexity) 노출 + 다국어(외국인 환자) 대응용
- 리스크: 같은 병원 정보가 두 도메인에 있으면 중복 콘텐츠로 검색엔진이 혼란스러워할 수 있음 → canonical 태그로 "이건 보조 콘텐츠"임을 명시 필요. 로코코 저널 실제 성과 데이터 있으면 그게 제일 강력한 영업 자료.

### 도메인 구조 (매우 중요 — 인덱싱 속도에 직접 영향)
- **서브도메인**(`journal.클리닉도메인.com`) 추천: 루트 도메인의 권위(domain authority)를 일부 물려받아 인덱싱/노출이 더 빠름
- **별도 도메인**으로 가면 신뢰도 0에서 시작 → 초기 노출 느림. 등록 여부보다 이 도메인 구조 선택이 훨씬 중요한 의사결정.

### 검색엔진 노출 = "메인 사이트 등록" 여부와 무관
- 구글/네이버 봇은 "메인이냐 세컨이냐" 구분 안 함 — 크롤링 가능하면 독립적으로 인덱싱됨. 메인 사이트로 지정 안 해도 검색에는 문제없음.
- 다만 Search Console에는 (메인이 아니어도) 별도 사이트로 등록 권장 — sitemap 제출로 인덱싱 속도 빨라지고 노출수/클릭수 데이터 확보 가능. "등록"은 모니터링 목적이고 메인/세컨 구분과는 별개.

### 가격 단가
- JS 없는 정적 사이트라고 가격 깎으면 안 됨 — 오히려 정적 구조가 AEO/크롤링엔 더 유리함. 파는 건 "사이트"가 아니라 "AEO/SEO 전략 + 구조화 데이터 설계 + 지속 모니터링"이므로 코드 복잡도가 아니라 전략 가치로 가격 책정해야 함.
- 단순 정적 사이트 시장가는 50만~150만원 수준(이 가격대로 묶이면 저렴해 보일 위험). 메디컬 대상 SEO 컨설팅/구축은 구축비 150만~400만원 + 월 유지비 20만~50만원 범위가 합리적 기준점(정확한 실시간 시장가 데이터는 아니므로 실제 견적 비교로 검증 필요).
- JS 없어 "느낌"이 허전해 보일 수 있으니 SEO에 영향 없는 가벼운 마이크로 인터랙션(스크롤 페이드인 등)만 추가하는 건 고려 가능 — progressive enhancement라 크롤링엔 영향 없음.

### 메인 홈페이지로 써도 되는 경우
- 기존 홈페이지 있는 병원: 무리 — 디자인 자산 투자 + 도메인 권한 리스크 때문에 세컨 사이트로 가는 게 맞음.
- 신규 오픈 병원/홈페이지 없는 소상공인·전문직(변호사/세무사): 이 구조 그대로 메인 사이트로 사용 가능. 오히려 이미지 통짜 사이트보다 검색엔진에 훨씬 유리한 시작점.

## 보류/병행 이슈

- **펫리뷰(petreview) 서비스**: 중단 결정. 단, 당장 강제로 끌 필요는 없음(무료 티어, 비용 없음). 신경 쓰인다면 Supabase 유저 데이터(가입자/리뷰/영수증) export 후 삭제 권장. 신규가입 막는 것도 옵션.
- **GitHub PAT 노출 사건**: 작업 중 실수로 journal/petreview의 GitHub PAT가 대화에 평문 노출됨. 로컬에만 있던 토큰이라 즉각 위험은 낮으나, 찜찜하면 재발급 권장.
- **모바일 앱 리포지토리 노출 문제**: `biniare-del`(개인) repo는 모바일에 보이는데 `rococops`(조직) repo는 안 보임 — GitHub App이 rococops 조직에 설치/권한 안 되어 있을 가능성. 본인이 조직 admin이면 github.com/settings/installations 에서 추가 설치 필요. 미해결 상태.
