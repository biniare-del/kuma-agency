## 핵심 영업/가격 전략 (중요)

### "세컨 홈페이지" 포지셔닝
- 기존 홈페이지(이미지/브랜딩 위주) vs 새로 만드는 사이트(검색엔진·AI검색 노출 전용)를 **명확히 구분해서 설명**해야 함. 안 그러면 고객이 "왜 두 개 필요하냐" 혼란스러워함.
- 멘트 기준: 기존 사이트 = 브랜딩/예약전환용, 새 사이트 = 검색엔진+AI검색(ChatGPT/Perplexity) 노출 + 다국어(외국인 환자) 대응용
- 리스크: 같은 병원 정보가 두 도메인에 있으면 중복 콘텐츠로 검색엔진이 혼란스러워할 수 있음 → canonical 태그로 "이건 보조 콘텐츠"임을 명시 필요. 로코코 저널 실제 성과 데이터 있으면 그게 제일 강력한 영업 자료.

### 도메인 구조 (매우 중요 — 인덱싱 속도에 직접 영향)
- **서브도메인**(`journal.클리닉도메인.com`) 추천: 루트 도메인의 권위(domain authority)를 일부 물려받아 인덱싱/노출이 더 빠름
- **별도 도메인**으로 가면 신뢰도 0에서 시작 → 초기 노출 느림. 등록 여부보다 이 도메인 구조 선택이 훨씬 중요한 의사결정.
- **호스팅 운영 방식 (확정)**: 클라이언트의 기존 메인 호스팅 계정(FTP/관리자 비밀번호 등)은 절대 받지 않음. 사이트 자체는 본인이 통제하는 호스팅(GitHub Pages/Vercel/Netlify 등)에 올리고, 클라이언트에게는 **DNS 관리 화면에 CNAME 레코드 1개 추가**만 요청(예: `journal.클리닉도메인.com` → 본인 호스팅 주소). 메인 사이트 파일/설정은 전혀 건드리지 않음 — 보안 책임 리스크 최소화 + "비밀번호 안 받는다"는 점 자체를 영업 시 신뢰 포인트로 활용 가능.
- **호스팅 플랫폼 선택 (확정)**: GitHub Pages는 저장소 1개당 커스텀 도메인 1개만 지원해 클라이언트가 늘어나면 매번 새 repo가 필요함 → 본인 운영용으로는 **Vercel 또는 Netlify(Cloudflare Pages도 동급)** 추천, 한 계정에서 프로젝트별 무제한 커스텀 도메인 연결 가능. 정적 사이트라 무료 티어로 클라이언트 수십 곳까지 비용 없이 운영 가능(유료 전환 시점은 트래픽 매우 커질 때, Pro 플랜 보통 월 $20대). **카페24 등에서 별도 "웹호스팅" 상품을 구매할 필요 없음** — 워드프레스/PHP/DB 기반이 아닌 정적 HTML/CSS라서 그런 서버가 불필요. 카페24가 등장하는 유일한 시나리오는 클라이언트의 기존 도메인이 카페24에서 관리되고 있을 때, 그 DNS 화면에 CNAME 레코드를 추가하는 작업뿐(구매/결제 없음).

### 검색엔진 노출 = "메인 사이트 등록" 여부와 무관
- 구글/네이버 봇은 "메인이냐 세컨이냐" 구분 안 함 — 크롤링 가능하면 독립적으로 인덱싱됨. 메인 사이트로 지정 안 해도 검색에는 문제없음.
- 다만 Search Console에는 (메인이 아니어도) 별도 사이트로 등록 권장 — sitemap 제출로 인덱싱 속도 빨라지고 노출수/클릭수 데이터 확보 가능. "등록"은 모니터링 목적이고 메인/세컨 구분과는 별개.

### 가격 단가
- JS 없는 정적 사이트라고 가격 깎으면 안 됨 — 오히려 정적 구조가 AEO/크롤링엔 더 유리함. 파는 건 "사이트"가 아니라 "AEO/SEO 전략 + 구조화 데이터 설계 + 지속 모니터링"이므로 코드 복잡도가 아니라 전략 가치로 가격 책정해야 함.
- 단순 정적 사이트 시장가는 50만~150만원 수준(이 가격대로 묶이면 저렴해 보일 위험). 메디컬 대상 SEO 컨설팅/구축은 구축비 150만~400만원 + 월 유지비 20만~50만원 범위가 합리적 기준점(정확한 실시간 시장가 데이터는 아니므로 실제 견적 비교로 검증 필요).
- **가격 확정 (2026-06-24)**: 처음 잡았던 150만원 베이스는 6개 진료과 풀세트(영문/후기/어드민/구조화데이터/AEO 대응) 수준에 비해 너무 낮다고 판단 → **정가 4,400,000원**으로 상향, 단 **"초기 레퍼런스 고객 한정 50% 할인"**을 내세워 실제 청구가는 **2,200,000원**으로 운영(레퍼런스 0개 상태에서 빠르게 사례 확보 목적, 정가/할인가 구조를 견적서·제안서에 항상 함께 노출해 "원래는 비싸다"는 인식을 줌). 정식가(할인 종료 후) 기준은 350만~440만원 구간. 외국어 페이지(영어 포함) 1개당 가격은 300,000원~500,000원 구간으로 상향(기존 20만~30만 → 40만원 기준으로 통일). `/quote/`와 `/proposal/`에 모두 반영 완료.
- JS 없어 "느낌"이 허전해 보일 수 있으니 SEO에 영향 없는 가벼운 마이크로 인터랙션(스크롤 페이드인 등)만 추가하는 건 고려 가능 — progressive enhancement라 크롤링엔 영향 없음.

### 메인 홈페이지로 써도 되는 경우
- 기존 홈페이지 있는 병원: 무리 — 디자인 자산 투자 + 도메인 권한 리스크 때문에 세컨 사이트로 가는 게 맞음.

### 꿈을담은에이전시 포트폴리오 1번에 반영할 사항 (확정)
1. **콘텐츠**: 로코코의 869개 동적 페이지 구조는 그대로 따라할 필요 없음(콘텐츠 자체가 차별점이었음) — 케이스/후기 통합페이지 1개만 추가
2. **상담 시스템**: 폼 UI는 유지. 실제 백엔드(Vercel/Resend/Supabase)는 포트폴리오 단계에서 구현 안 함. 카카오채널 플로팅 버튼 UI 추가
3. **다국어**: 랜딩 1개(영어)만 실제 제작 + 헤더에 **다국어 국기 아이콘**을 보여줘서 "다양한 언어 대응 가능"임을 시각적으로 어필 (나머지 언어는 깃발만 노출, 실제 페이지는 없음)
4. **SEO/AEO/GEO — 로코코보다 더 정교하게 만들 포인트** (전부 반영 추천):
   - hreflang 태그 (로코코는 다국어 페이지에 이게 빠졌을 가능성)
   - Physician(의사 개인) schema
   - AggregateRating/Review schema
   - Speakable schema (음성/AI 답변 인용용)
   - robots.txt에 GPTBot/PerplexityBot/ClaudeBot 등 AI 크롤러 User-Agent 명시적 Allow
   - 시술 페이지 상단 TL;DR 요약 블록 (AI 발췌·인용 최적화, GEO 핵심 기법)
   - Core Web Vitals 증거(이미지 lazy-load, WebP 등) — 로코코는 핫링크 의존이라 약점
5. **어드민 패널**: 포트폴리오 단계에서는 실제 백엔드(인증/DB/실시간 분석) 구현 안 하고 **비주얼 완성도 높은 UI 목업**으로 제작. 사용자 요청 반영 내용:
   - 로그 분석 대시보드 (노출수/클릭수/인기 시술 등 차트·그래프 시각화, 가짜 데이터)
   - 콘텐츠 추가/관리 페이지 (CMS형)
   - **회원가입/로그인 게이트**: 전후사진은 의료법상 비회원 열람 제한 필요 → 로그인 UI 필수로 설계 (실제 인증 로직은 미구현, 영업 자료용 UI)
- 위 1~4번까지는 사용자 확정. 5번(어드민 UI 목업 방향)도 사용자 동의, 실제 구현 착수 전 상태.
